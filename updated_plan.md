CardCity Project
Revised Plan

With all intended features implemented and sufficiently tested, I am quite proud of where the project is at. There are but a few aspects I will be working to improve in the coming week. 

Firstly, I will continue cleaning up the code. Some JavaDoc comments are unclear, some methods are unnecessarily long or redundant, and some code could become more efficient. Going through different classes and finding these imperfections, making sure the code is readable and proper is my number one priority now that the game is fully playable.

Secondly, I will investigate text-scaling so that all text is visible and not overlapping after resizing the “Stats!” window. This, however, is lower on my priorities as it is not a major issue.

Thirdly, I will take feedback from my peers during my presentation and work to think about and implement their ideas, if said ideas fit within the scope and idea of the game and the time I have left to work on it.

Furthermore, the game is playable after losing, the stats simply do not update. I will fix this so that the game is no longer interactable once the game ends.

Lastly, I will continue rigorously playtesting the game, fixing bugs I come across and balancing the abilities of cards in the game.
